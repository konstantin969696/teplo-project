## Требования к экспорту (от Кости 2026-04-26)

**Обязательные:**
- Все три формата на выбор: **Excel**, **Word**, **PDF**.
- Каждая страница — **с ГОСТ-рамкой** (ГОСТ 2.104-2006, основная надпись форма 1).
- **Штамп** — параметры заполняются в UI перед экспортом (форма-окошко с полями «Разработал», «Проверил», «Утвердил», «Шифр объекта», «Стадия», «Лист N из M», «Наименование», «Заказчик» и т.д.). Значения сохраняются в `projectStore` чтобы не вводить заново.
- **Форматы листа на выбор:** A1, A2, A3, A4 (ГОСТ 2.301-68), плюс **склейки A3×3, A3×4, A2×2 и т.д.** для длинных таблиц/планов. Список расширяемый.
- **Предпросмотр в отдельном модальном окне** перед экспортом — пользователь видит листы как они выйдут (с рамкой, штампом, контентом), пролистывает страницы, потом нажимает «Экспорт».
- Весь UI экспорта (кнопки, поля, подсказки, ошибки, имена файлов по умолчанию) — **на русском**.

**Архитектурное следствие:** общая модель «лист с рамкой и штампом» делается один раз в фазе 06. Каждый экспортёр (PDF/Excel/Word) — это backend-адаптер поверх этой модели. Это упрощает и тестирование (рамка/штамп тестируются единожды), и дальнейшее расширение (новый формат — новый адаптер, не дубль рамок).
