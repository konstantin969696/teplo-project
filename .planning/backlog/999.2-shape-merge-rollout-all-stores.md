# 999.2 — Раскатать shapeMerge на остальные persist-stores (бэклог)

**Origin:** обнаружено при закрытии фазы 04.2.

**Что:** в фазе 04.2 защита `shapeMerge` (валидация persisted shape) добавлена только в 4 stores: `systemStore`, `segmentStore`, `equipmentStore`, `ufhLoopStore`. Остальные persist-stores (`enclosureStore`, `constructionStore`, `pipeCatalogStore`, `coolantCatalogStore`, `kmsCatalogStore`, `catalogStore`) не защищены — если у них в `localStorage` тоже окажется поле неверного типа, та же поломка повторится.

**Что сделать:**
1. Профилактически добавить `merge: shapeMerge(...)` во все persist-stores проекта.
2. Для каждого — отдельный test-файл вида `*.persist.test.ts` (по аналогии с `systemStore.persist.test.ts`).

**Объём:** ~30 минут на store × 6 = ~3 часа.

**Когда брать:** после M1, либо если поймаем регрессию по другому store. Не блокирующий релиз.
