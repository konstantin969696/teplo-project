# GOST-EXPORT-SPEC -- Экспорт документации по ГОСТ (PDF / Excel / Word)

Цель: расширить существующий конвейер экспорта `src/export/` выводом в три формата -- PDF, Excel (.xlsx) и Word (.docx) -- с оформлением по ГОСТ Р 21.101-2020, ГОСТ 2.105-2019 и ГОСТ 21.110-2013. В текущем коде уже реализован PDF-backend на jsPDF с ГОСТ-рамкой (фаза 07), штампом формы 1 (фаза 07.1), пагинацией (`layout.ts`), format-agnostic билдерами (`builders.ts`) и persist-store (`exportStore.ts`). Настоящая спека добавляет Excel/Word бэкенды, корректный штамп для текстовых документов (форма 5 / форма 6) и UI формы штампа.

---

## Нормативная база

| Стандарт | Что берём |
|---|---|
| **ГОСТ Р 21.101-2020** (СПДС) | Формы основной надписи: форма 5 -- первый (заглавный) лист текстовых документов; форма 6 -- последующие листы. Рамка и поля -- приложение И. Боковая полоса (графы 19--23) -- слева на поле подшивки. |
| **ГОСТ 2.105-2019** (ЕСКД) | Поля текстового документа: левое 20 мм, правое 5 мм, верхнее 5 мм, нижнее 5 мм. Расстояние от текста до рамки не менее 10 мм. Абзацный отступ 12,5--17 мм. |
| **ГОСТ 21.110-2013** (СПДС) | Спецификация оборудования -- форма 1, колонки: позиция, наименование и техническая характеристика, тип/марка/обозначение, количество, примечание. Выполняется на А3 (или А4 при модификации). |

## Штамп (основная надпись)

### Форма 5 -- первый лист текстового документа

Размеры штампа: **185 × 40 мм** (в отличие от формы 1 для чертежей -- 185 × 55 мм). Располагается в правом нижнем углу листа. На А4 -- вдоль короткой стороны.

| Графа | Размер (мм) | Содержание | Источник данных |
|---|---|---|---|
| 1 -- Наименование | 185 × 15 (верх штампа) | Название документа, например «Расчёт теплопотерь» | `stamp.drawingTitle` |
| 2 -- Обозначение | 185 × 10 (под наименованием) | Шифр документа | `buildDesignationCode(stamp)` |
| 3 -- Материал | -- | Не заполняется для текстовых документов | -- |
| 4 -- Литера | 15 × 5 | Стадия: П / Р / РД / Э | `stamp.stageCode` |
| 5 -- Масса | -- | Не заполняется | -- |
| 6 -- Масштаб | -- | Не заполняется для текстовых документов | -- |
| 7 -- Лист | 20 × 5 | Номер листа (авто) | `sheet.index + 1` |
| 8 -- Листов | 20 × 5 | Всего листов (авто, только на 1-м листе) | `sheet.total` |
| 9 -- Организация | 40 × 40 (правый блок) | `companyName` + `logoDataUrl` | `stamp.companyName` |
| 10 -- Должность | 17 × 6 × 5 строк | Разраб. / Проверил / ГИП / Н. контр. / Утв. | константы |
| 11 -- Фамилия | 20 × 6 × 5 строк | Фамилии с инициалами | `stamp.authorName` и т.д. |
| 12 -- Подпись | 15 × 6 × 5 строк | (пусто, место для подписи) | -- |
| 13 -- Дата | 13 × 6 × 5 строк | Дата подписания | `signerDate(stamp, role)` |
| 14--18 -- Изменения | 75 × 25 (нижний-левый блок) | Таблица изменений (заголовок + 4 пустые строки) | не заполняется при первой печати |
| 19--23 -- Боковая полоса | 7 × высота рамки (слева) | Согласовано / Инв.№ подп. / Подп. и дата / Взам. инв. № | `stamp.agreedBy` и т.д. |

Геометрия формы 5 (относительно левого верхнего угла штампа):

```
Ширина 185 мм, высота 40 мм

┌─────────────────────────────────────────────────────────────────┐
│  Графа 1 -- Наименование (185 × 15)                              │
├─────────────────────────────────────────────────────────────────┤
│  Графа 2 -- Обозначение (185 × 10)                               │
├──────────┬──────────┬──────────┬────────────────────────────────┤
│ 4 Литера │ 7 Лист   │ 8 Листов │ 9 Организация (40 × 40)        │
│ 15×5     │ 20×5     │ 20×5     │                                │
├──────────┴──────────┴──────────┤                                │
│ 10 Должность │ 11 Фамилия │ 12 Подп. │ 13 Дата │                │
│ 17×6         │ 20×6       │ 15×6     │ 13×6    │                │
│ 5 строк      │            │          │         │                │
└─────────────────────────────────────────────────────────────────┘
```

### Форма 6 -- последующие листы текстового документа

Размеры штампа: **185 × 15 мм** (усечённая форма без таблицы изменений и без графы наименования).

| Графа | Размер (мм) | Содержание |
|---|---|---|
| 2 -- Обозначение | 70 × 10 | Шифр документа (тот же, что на 1-м листе) |
| 7 -- Лист | 20 × 5 | Номер листа |
| 8 -- Листов | 20 × 5 | Всего листов (только на 2-м листе, если он считается "первым рабочим") |
| 9 -- Организация | 40 × 15 | Организация (усечённая) |
| 10--13 -- Подписи | 75 × 15 | Только должность/фамилия/подпись/дата (без заголовков колонок) |

На последующих листах графа 8 "Листов" может не заполняться -- общее количество уже указано на первом листе. В реализации оставляем её для единообразия.

## Архитектура

### Изменения в типах

**Решение:** `GostStampParams` как отдельный объект внутри `ProjectData`, независимый от `Stamp` (который используется для чертежей формы 1).

Обоснование:
- Текстовые документы (пояснительные записки, спецификации) используют форму 5/6, а чертежи -- форму 1. Поля различаются (например, в форме 5 нет графы "Масштаб", зато есть "Литера").
- Разделение типов позволяет валидировать и рендерить форму штампа отдельно для каждого режима.
- `Stamp` (существующий) остаётся без изменений для backward-compat существующих PDF-экспортов чертежей.

```ts
// src/export/types.ts -- дополнения

export type TextDocStampMode = 'full' | 'minimal-footer' | 'none'

export interface GostStampParams {
  readonly objectName: string           // графа 1 -- наименование объекта
  readonly objectCode: string           // шифр объекта: "70-2025"
  readonly subsectionCode: string       // подраздел: "ИЛО", "ПЗ"
  readonly stageCode: string            // 'П' | 'Р' | 'РД' | 'Э'
  readonly markCode: string             // марка: 'ОВ', 'ВК'
  readonly drawingTitle: string         // название документа
  readonly drawingMark: string          // марка чертежа (override шифра)

  readonly authorName: string
  readonly checkerName: string
  readonly gipName: string
  readonly normControlName: string
  readonly approverName: string

  readonly companyName: string
  readonly companyDept: string
  readonly logoDataUrl?: string

  readonly date: string                 // ISO YYYY-MM-DD
  readonly signDates?: {
    readonly author?: string
    readonly checker?: string
    readonly gip?: string
    readonly normControl?: string
    readonly approver?: string
  }

  readonly agreedBy?: string
  readonly inventoryNumber?: string
  readonly replacedInventoryNumber?: string
}

export interface DocumentModel {
  // ... существующие поля без изменений ...
  readonly gostStamp?: GostStampParams  // для текстовых документов (форма 5/6)
}
```

### Выбор библиотек

#### PDF -- оставляем jsPDF

| Библиотека | Кириллица | Точные мм | ГОСТ-рамки | Обоснование |
|---|---|---|---|---|
| **jsPDF** | ✅ (TTF base64 embed) | ✅ (`unit: 'mm'`) | ✅ (прямые `rect`/`line`) | Уже в проекте, проверен в фазах 07/07.1. Font-loader (`fontLoader.ts`) кэширует base64. |
| pdf-lib | ✅ (fontkit + embed) | ✅ (pt→mm) | ✅ | Более современный API, но требует `@pdf-lib/fontkit`. Переход даёт мало выигрыша -- jsPDF уже работает. |
| pdfmake | ✅ (vfs_fonts.js) | ⚠️ (declarative, менее точный) | ⚠️ | Тяжёлый vfs для кириллицы (~1,5 МБ). Не даёт pixel-perfect контроля линий. |

**Победитель: jsPDF** -- остаём на нём. Добавляем `drawStampForm5` и `drawStampForm6` рядом с существующим `drawStampForm1`.

#### Excel -- SheetJS (xlsx)

| Библиотека | Браузер | Стили | Размер | Обоснование |
|---|---|---|---|---|
| **SheetJS (xlsx)** | ✅ | ⚠️ (базовые через `cell.s`) | ~500 КБ | Стандарт индустрии, работает в браузере без Node.js. Достаточно для ГОСТ-спецификации (границы, шрифт, выравнивание). |
| ExcelJS | ✅ (browser bundle) | ✅ (богатые) | ~1,2 МБ | Требует `buffer` polyfill в браузере. Стили лучше, но избыточны для задачи. |
| xlsx-populate | ✅ | ✅ | ~400 КБ | Меньше community, рискованнее для долгосрочной поддержки. |

**Победитель: SheetJS (xlsx)** -- `npm install xlsx`. Для ГОСТ-спецификации достаточно: колонки с ширинами, границы ячеек (`border`), жирный шрифт (`bold`), объединённые ячейки (`merge`).

#### Word -- docx (npm package `docx`)

| Библиотека | Браузер | Таблицы | Шаблоны | Обоснование |
|---|---|---|---|---|
| **docx (npm)** | ✅ | ✅ (programmatic) | ❌ (кодом) | Работает в браузере, TypeScript-first, активно поддерживается. Позволяет строить DOCX pixel-perfect через `Table`, `Paragraph`, `BorderStyle`. |
| docxtemplater | ✅ | ✅ (via template) | ✅ | Требует заранее подготовленный .docx-шаблон. Для ГОСТ-рамки проще сгенерировать таблицу-каркас кодом, чем поддерживать шаблон. |
| easy-template-x | ✅ | ✅ | ✅ | Та же проблема -- нужен шаблон. |

**Победитель: docx (npm package)** -- `npm install docx`. Строим рамку как `Table` с точными ширинами колонок (в twips), штамп -- как отдельная `Table` внизу страницы.

## UI

### Модалка экспорта (существующая)

Дополнить существующий диалог экспорта:

1. **Селектор формата** -- радио-кнопки: PDF / Excel / Word.
2. **Селектор типа документа** -- выпадающий список: Теплопотери / Приборы отопления / Гидравлика / Тёплый пол / Сводка / Спецификация оборудования.
3. **Селектор формата листа** -- A4 / A3 / A3×3 (для спецификаций).
4. **Форма штампа** -- раскрывающаяся панель:
   - Шифр объекта (objectCode)
   - Подраздел (subsectionCode)
   - Стадия (stageCode) -- выпадашка: П, Р, РД, Э
   - Марка (markCode) -- выпадашка: ОВ, ВК, ТХ, ЭО, СС
   - Название документа (drawingTitle)
   - Марка чертежа (drawingMark) -- опц.
   - Разработал / Проверил / ГИП / Н. контр. / Утв. -- строки
   - Организация / Отдел -- строки
   - Логотип -- загрузка файла → `FileReader` → `dataUrl`
   - Дата -- `input type="date"`
5. **Превью** -- миниатюра первой страницы (SVG или canvas 150×200 px) с рамкой и штампом. Обновляется при изменении любого поля (debounce 300 мс).
6. **Кнопки** -- "Экспортировать" (primary) / "Отмена".

### Валидация

- `objectCode` -- обязательное, маска `\d{2}-\d{4}` (или свободный ввод с подсказкой).
- `stageCode` -- обязательное, одно из `['П','Р','РД','Э']`.
- `authorName`, `checkerName` -- обязательные (остальные подписанты опц.).
- `date` -- обязательная, формат `YYYY-MM-DD`.

## Граничные случаи

| Условие | Поведение |
|---|---|
| Пользователь не заполнил обязательные поля штампа (objectCode, stageCode, authorName) | Кнопка "Экспортировать" disabled, под полем красная подсказка "Обязательное поле". |
| Документ из 1 листа | Форма 5 на единственном листе, графа 8 "Листов" заполнена. Форма 6 не используется. |
| Спецификация пустая (нет оборудования) | Excel/Word: выводится шапка + строка "Спецификация не содержит позиций". PDF: пустая таблица с шапкой. |
| Формат A3×3, но контент умещается на 1 страницу A3 | Пользователь сам выбирает формат. Пагинация не добавляет лишних страниц. |
| Название документа (drawingTitle) длиннее 185 мм | В штампе формы 5: `truncateToWidth()` с многоточием. Максимум 2 строки по 7 мм, затем обрезка. |
| Логотип не загружен | Правый блок штампа (графа 9) заполняется только `companyName`, линия разделения лого/текст не рисуется. |
| Логотип > 2 МБ (base64) | UI предупреждает "Логотип слишком большой, уменьшите до 2 МБ". Принудительно сжимается через `HTMLCanvasElement.toDataURL('image/jpeg', 0.7)` до достижения лимита. |
| Дата в неправильном формате | UI использует `input type="date"` → браузер гарантирует `YYYY-MM-DD`. В штампе рендерится как `DD.MM.YYYY`. |
| Excel без штампа | Штамп в Excel не рендерится (формат не позволяет pixel-perfect). Вместо этого -- строка метаданных над таблицей. |
| Word на нескольких страницах | Штамп формы 5 только на первой странице; форма 6 на последующих. Рамка -- на каждой странице через `Header` с таблицей-рамкой (если `docx` API позволяет) или через `Section` с повторяющимся `Header`. |
| Браузер не поддерживает `input type="date"` | Fallback на текстовый ввод с regex валидацией `^\d{4}-\d{2}-\d{2}$`. |
| Отсутствует `gostStamp` в `DocumentModel` | Fallback на существующий `stamp` + форму 1 (чертежный режим). Backward-compat сохранён. |

## Из scope ИСКЛЮЧЕНО

- **Экспорт в AutoCAD (.dwg)** -- отдельная задача, требует серверного конвертера или ODA File Converter.
- **Электронная подпись (ЭП)** -- не входит в MVP, требует КриптоПро или аналогов.
- **Штрих-коды в штампе** -- ГОСТ Р 21.101-2020 п. 5.2.9 допускает, но не требует.
- **Титульный лист по приложению Р** -- отдельный документ, не часть листа со штампом.
- **Журнал изменений (приложение П)** -- не требуется для первой версии.
- **ГОСТ-шрифт 2.304 тип А** -- в текущем коде используется DejaVu Serif как заместитель. Полноценный GOST 2.304 требует лицензированного TTF, который не входит в open-source.
- **Пагинация в Word/Excel** -- Excel не paginated, Word полагается на рендеринг MS Word при открытии.

## Контракт для исполнителя (Саске)

1. **Атомарные коммиты** -- одна фаза = один коммит. 4 коммита всего, ветка `feature/gost-export`.
2. **Тесты обязательны** -- каждый бэкенд (PDF/Excel/Word) имеет минимум 1 юнит-тест на возврат Blob с правильным MIME. Геометрия штампа -- юнит-тест на сумму размеров. `npm test -- --run` зелёное перед каждым коммитом.
3. **Не трогай** -- `engine/heatLoss.ts`, `engine/hydraulics.ts`, `engine/ufh.ts`, `store/projectStore.ts` (кроме типов). Только `src/export/` и `src/components/export/`.
4. **Backward-compat** -- существующие PDF-экспорты чертежей (форма 1) должны работать без изменений. Поле `gostStamp` в `DocumentModel` -- опциональное.
5. **Перед коммитом каждой фазы:**
   - `npm test -- --run` зелёное.
   - `npx tsc --noEmit` чисто.
   - `npm run build` успешно (проверка tree-shaking и Vite bundle).
6. **Ветка:** `feature/gost-export`. PR в `main` после Phase 4.
